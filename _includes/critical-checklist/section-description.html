## {{ title }} These tests check to see if the accordion is accessible and usable {{ description }}. For more instructions on how to perform keyboard control tests, check out the resources in the expandable section below.

Through this process of screen reader testing, you can confirm the accordion component complies with:

  • 2.1.1 Keyboard: All functionality of the accordion is operable through a keyboard interface. (Level A)
  • 2.1.2 No Keyboard Trap: If keyboard focus can be moved to a component of the page using a keyboard interface, then focus can be moved away from that component using only a keyboard interface. (Level A)
  • 2.4.3 Focus Order: Focusable elements receive focus in an order that preserves meaning and operability. (Level A)
  • 2.4.7 Focus Visible: The keyboard focus indicator is visible. (Level AA)
